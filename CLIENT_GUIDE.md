# 運営者向け簡易マニュアル

カジノBotの**運営者**(=管理者)向けガイドです。プレイヤー向けの遊び方は `/カジノ → ❓ ルール` を見てもらってください。

## 自分が管理者になっているか確認
`/管理` を叩いて開けばOK。`🚫 管理者専用です。` と返るなら、運営に依頼してDB管理者として追加してもらってください。

## いまどのカテゴリで何ができる？

`/管理` を叩くと **5カテゴリのメインメニュー** が出ます。

### 👤 ユーザー操作 (日常運用)
- **➕ 残高 付与** — 特定ユーザーにチップを配る
- **➖ 残高 没収** — チップを取り戻す
- **↩️ 直前の操作を取消** — あなたの最後の操作1件を取り消す
- **🧊 凍結** / **☀️ 解凍** — 不正ユーザーや退会者を停止/再開
- **🔍 取引履歴 監査** — そのユーザーの履歴一覧

> 残高変更は必ず **確認ステップ** を挟みます。10万以上は **理由必須**、自分への付与は **禁止**、5秒CDあり。

### 💰 経済 (バランス調整・イベント)
- **📊 経済ダッシュボード** — Gini係数、月次インフレ率、健康度判定
- **📋 設定一覧** / **🛠️ 設定を変更** — daily額、ハウスエッジ、レーキ率、ガチャ価格 等の調整
- **🏆 大会を開催** — 期間/賞金/種類(収支/連勝/JP)を指定
- **🚀 ブースト開始** / **🛑 ブースト終了** — 期間限定で全PVE配当倍率アップ
- **📢 お喋りCH をここに** — 押下したチャンネルが「JP当選などの告知先」になる

### 💱 両替
- **📥 承認CHをここに設定** — 両替申請の承認ボタンを送る運営専用チャンネル
- **🔥 お釈迦さま設定** — ゼニーを焼却する受け取りアカウントID
- **📋 両替申請一覧** — 保留中の申請を一覧

### 🛠️ システム
- **🛠️ メンテモード切替** — 一般プレイ停止/再開(自動アナウンス)
- **🔄 Cogリロード** — 個別機能を無停止で差し替え

### 🚨 危険ゾーン (env管理者のみ)
- **🎯 残高 セット** — 直接書き換え(差分は監査ログに)
- **👥 管理者一覧** — env由来 / DB由来 を区別表示
- **➕ 管理者追加** — 新しい運営者を即時登録
- **➖ 管理者削除** — env由来は弾く

> このゾーンには **`.env` 由来の初期管理者だけ** 入れます。DB管理者はここのボタンを押せません(権限拡散防止)。

## よくあるシチュエーション別 操作

### Q. 新しい運営スタッフを管理者にしたい
1. その人の Discord ユーザーID を教えてもらう (右クリック→ユーザーIDをコピー)
2. `/管理` → 🚨危険ゾーン → ➕管理者追加 → ID入力
3. 即時反映

### Q. 両替申請が来てるけど運営チャンネルが分からない
1. 運営専用チャンネルを1つ作る (例: `#両替承認ログ`、運営のみ閲覧可推奨)
2. **そのチャンネル内で** `/管理` → 💱両替 → 📥承認CHをここに設定
3. 以降、申請は自動でそのチャンネルに承認/拒否ボタン付きで届く

### Q. お喋りログを派手にしたい(JP当選などの告知)
1. プレイヤーが見えるチャンネルを用意 (例: `#カジノ実況`)
2. **そのチャンネル内で** `/管理` → 💰経済 → 📢お喋りCH をここに

### Q. インフレが心配
1. `/管理` → 📊経済ダッシュボード → 月次純発行と Gini をチェック
2. 🟡や🔴が出てたら 💰経済 → 設定を変更 から：
   - `slot_house_edge` を 0.05→0.07
   - `daily_decay_pivot` を下げる(=金持ちのデイリーをもっと絞る)
   - `holding_tax_enabled` を 1 に (退蔵に税金)

### Q. 大会を開きたい
1. `/管理` → 💰経済 → 🏆大会を開催
2. 種類選択 (💰収支 / 🔥連勝 / 💎JP獲得)
3. 期間(時間)・賞金プール額・任意の大会名を入力
4. お喋りログに自動告知、参加登録は不要(プレイしてれば自動エントリー)

### Q. 怪しいユーザーを止めたい
1. `/管理` → 👤ユーザー操作 → 🧊凍結 → 対象IDを入力
2. 凍結中は賭博・両替・送金が全てブロック
3. 確認できたら ☀️解凍 で復帰

### Q. メンテしたい
1. `/管理` → 🛠️システム → 🛠️メンテモード切替 ON
2. 作業
3. もう一度 ON→OFF

メンテ中も管理者は触れます(動作確認可能)。

## 何かトラブった時

- **Bot がオフライン** → 開発担当(あなたや僕)に連絡
- **`/管理` が「管理者専用」と言われる** → env管理者に追加してもらう
- **「インタラクションに失敗」** → 同じ操作をもう一度試す。連発するなら開発担当に
- **残高がおかしい** → 監査でログ確認、必要なら 取消 or 残高セット (env管理者)

例外発生時は **自動的に env管理者全員にDMで通知** されます。何かあればDM確認を。
